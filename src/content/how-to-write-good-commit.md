我印象非常深刻發生在我第一份工作所待的團隊，還記得那天第一天上班裝好 Source Tree 準備來看專案的 git log，一打開真的嚇到我了，這個團隊的 commit message 太精彩了，什麼亂碼都有甚至還有 mesage 打"fuck you"的。
對於一個新人的我而言，看到這種 commit message 完全無法理解他們最近到底做了什麼或是哪一段程式碼為什麼要這樣改，我兩個月後真的受不了這種沒規範的寫法就跑了。

## commit message 為什麼重要

『commit message 是你與團隊成員以及未來的自己溝通的主要橋樑之一』
我們都有可能會需要改動團隊成員曾經寫過的 code 或是回頭改自己之前寫的 code，一個好的 commit message 能清楚地幫助你知道為什麼當初這個改動是為了什麼。
